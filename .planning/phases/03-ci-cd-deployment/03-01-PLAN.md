  <behavior>
    --- Workflow Trigger ---
    - Workflow triggers on schedule: `0 6 * * 1` (Monday 06:00 UTC)
    - Workflow triggers on `workflow_dispatch` (manual trigger via GitHub UI)
    - No other triggers (no push, no PR)

    --- Workflow Permissions ---
    - `contents: write` — for git commit/push of data.json and checkout
    - `pages: write` — for deploy-pages action
    - `id-token: write` — for deploy-pages OIDC authentication

    --- Concurrency ---
    - Group name: `pages`
    - `cancel-in-progress: false` — never cancel an in-progress run

    --- Job 1: build ---
    - Runs on: ubuntu-latest
    - Steps:
      1. `actions/checkout@v4`
      2. `actions/setup-go@v5` with `go-version-file: go.mod` and `cache: true`
      3. `go run ./cmd/meters fetch` — downloads xlsx. Non-zero exit propagates to workflow failure.
      4. `go run ./cmd/meters build` — parses xlsx and writes data.json. Non-zero exit propagates to workflow failure.
      5. Conditional commit step — shell script:
         - Use `git diff --quiet data.json` to check if data.json changed
         - If exit code != 0 (data changed): configure git user.name as `github-actions[bot]`, user.email as `github-actions[bot]@users.noreply.github.com`, then `git add data.json`, `git commit -m "chore: refresh data.json"`, and `git push`
         - If exit code == 0 (data unchanged): output "data.json unchanged, skipping commit" — no error, no-op
      6. Prepare deploy artifact:
         - `mkdir -p _site`
         - `cp index.html data.json _site/`
         - `cp -r site _site/`
         - `touch _site/.nojekyll`
      7. `actions/upload-pages-artifact@v4` with `path: _site`

    --- Job 2: deploy ---
    - `needs: build` — automatically skipped if build job fails
    - Runs on: ubuntu-latest
    - `environment:` name `github-pages`, url from `${{ steps.deployment.outputs.page_url }}`
    - Steps:
      - `id: deployment`, uses `actions/deploy-pages@v4`

    --- .nojekyll ---
    - Empty file (0 bytes) at repo root
    - Committed to git so repo root serves correctly on GitHub Pages even without deploy artifact

    --- PROJECT.md ---
    - Update the CI/CD section to document GitHub Pages source configuration
    - Document that Pages source must be set to "GitHub Actions" in repo Settings (one-time manual step)
  </behavior>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions workflow -> git push | Workflow has contents:write permission. Conditional commit only modifies data.json. |
| Deploy artifact -> GitHub Pages | Artifact contains only static files. No user-submitted content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | workflow permissions | mitigate | Grant minimal permissions: contents:write (data.json only), pages:write, id-token:write |
| T-03-02 | Tampering | conditional commit | mitigate | Only data.json is staged and committed. git diff check prevents no-op commits |
| T-03-03 | Tampering | deploy artifact | mitigate | Artifact only includes predetermined static files. No runtime code or user input |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan. Zero external dependencies. Package Legitimacy Audit: N/A |
