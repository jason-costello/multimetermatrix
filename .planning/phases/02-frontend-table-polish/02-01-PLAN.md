  <implementation>
    Create site/engine.js with ES module exports. Each function is a pure function:

    - `escapeHtml(str)`: Replace &, <, >, " with HTML entities. Handle null/undefined → "".
    - `highlightMatch(value, query)`: HTML-escape value first, then if query is non-empty, build case-insensitive RegExp, wrap matches in `<mark class="search-match">`. Escape regex special chars in query via `query.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')`.
    - `parseNumericValue(value)`: Strip leading $, remove commas, trim whitespace, then match /^([\d.]+)([a-zA-ZμΩ°]*)?$/. Apply K=*1000, M=*1000000, m=*0.001, u=*0.000001, n=*0.000000001 multipliers. Return null for unparseable.
    - `getCellColors(band, flag, editionDate)`: If band equals editionDate, treat as none. Map band→color from lookup table. Map flag→color from lookup table. Band takes priority over flag. Return { bg: hex, text: "#ffffff" or "#1a1a2e" } based on luminance threshold 0.5 per UI-SPEC.
    - `filterRows(rows, filters)`: Apply each filter type (band, flag, numeric) sequentially with AND logic. Each is a pure predicate check. Return new filtered array. Per D-11: search AND sidebar filters (search handled separately in app.js, combined here).
    - `sortRows(rows, columnKey, direction)`: Return new sorted array. Use numeric comparison via parseNumericValue, fall back to String.localeCompare with numeric:true and sensitivity:'base'. Tiebreaker: compare original index (rows.indexOf). direction "none" returns [...rows].
    - `searchRows(rows, query)`: Return filtered array where at least one column value contains query (case-insensitive via toLowerCase + includes).
    - `getBandColumns(data)`: Return array of column names with 3+ distinct band values excluding edition date. Derived from analysis: the 29 columns listed in interfaces section.
    - `getFlagColumns(data)`: Return array of column names that have any flag data in rows.
    - `getNumericColumns(data)`: Return ["Price", "Count", "Yr"] (v1 scope per RESEARCH.md).

    Implementation must be minimal in GREEN phase — no premature optimization. Use simple array methods, not complex data structures. All functions use `for` loops or array methods; none use mutation on inputs.
