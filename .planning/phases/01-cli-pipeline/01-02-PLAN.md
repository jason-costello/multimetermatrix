    Create cmd/meters/color.go with package main.

    Define type `legendEntry struct { hex string; label string }` to carry a legend cell's fill color and label.

    Define type `sRGB` with fields R, G, B as float64 (0.0 to 1.0).

    Implement these exported functions:

    1. `hexToLinearRGB(hex string) (sRGB, error)`:
       - Expect 6 hex chars (no alpha "FF" prefix -- caller strips it)
       - Parse R, G, B as uint8 via strconv.ParseUint(hex[0:2], 16, 8) etc.
       - Convert each from [0,255] to [0.0,1.0], then apply linearizeSRGB
       - Return sRGB struct

    2. `linearizeSRGB(c float64) float64`:
       - IEC 61966-2-1 piecewise: if c <= 0.04045, return c / 12.92; else return math.Pow((c+0.055)/1.055, 2.4)
       - Use math.Pow for gamma expansion

    3. `euclideanDistance(a, b sRGB) float64`:
       - Return squared Euclidean distance (dr*dr + dg*dg + db*db)
       - No sqrt -- squared distance is sufficient for comparison

    4. `nearestBand(hex string, bands []legendEntry) (bestLabel string, tied bool)`:
       - Convert hex to linear sRGB
       - For each band entry, compute euclideanDistance
       - Track minimum distance and which label(s) achieve it
       - If multiple bands tie for minimum distance, set tied=true and assign the HIGHER band per D-07
       - "Higher band" means the ordering from the legend row position: earlier in the row = higher (V High > High > Average > Low > V Low). Use slice index -- lower index = higher band.
       - If hex is empty, return ("", false)

    5. `isCategoricalMarker(hex string, markers []legendEntry) (label string, exact bool)`:
       - Compare hex string (case-insensitive) against each marker's hex
       - Exact match required per D-08
       - Return matching label and true, or ("", false)
       - Strip alpha "FF" prefix from both hex and marker hex before comparison

    6. `stripAlpha(hex string) string`:
       - If len(s) == 8 && strings.HasPrefix(s, "FF"), return s[2:]
       - Otherwise return s as-is

    All functions are pure math/string -- no I/O, no dependencies on other CLI packages.
    No fenced code blocks in action. Derive exact implementation from the RESEARCH.md code examples.
  </action>
