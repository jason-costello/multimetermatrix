  <action>
    Create cmd/meters/fetch.go with:

    Package main. Imports: fmt, os, net/http, io, time, strings, path/filepath.

    - `const defaultExportURL = "https://docs.google.com/spreadsheets/d/1JB1xLWaXLOCWfANM1O_2Vgg_KKbl0wTQCi409aIV6Jg/export?format=xlsx"` (from PROJECT.md line 11)

    - `func fetchCmd(args []string)`: 
      - Create `flag.FlagSet` for "fetch" subcommand
      - Register `--url` flag with default value of defaultExportURL
      - Register `--verbose` / `-v` bool flag (false default)
      - Parse args; if parse error, os.Exit(2)
      - Call `runFetch(url, verbose)`
      - If error, print error message to stderr and os.Exit with appropriate code per D-15

    - `func runFetch(url string, verbose bool) error`:
      - Create http.Client with 30s timeout
      - GET the URL
      - On error: classify error message
        - If timeout-related: return exit-code-3 error with prefix "FETCH_TIMEOUT"
        - If DNS/hostname error: return exit-code-3 error with prefix "FETCH_DNS_FAILURE"
        - Other: return exit-code-1 error with prefix "FETCH_FAILED"
      - Defer resp.Body.Close()
      - Check HTTP status:
        - 404: return exit-code-3 error with prefix "FETCH_404" including the URL
        - 5xx: return exit-code-3 error with prefix "FETCH_5xx" including status code
        - Not 200: return exit-code-3 error with prefix "FETCH_HTTP_{code}"
      - Read resp.Body into []byte
      - Check first 2 bytes == "PK" (ZIP magic bytes). If not, return error "FETCH_BAD_MAGIC: ..."
      - Check len(data) >= 10240 (10KB). If not, return error "FETCH_TOO_SMALL: ..."
      - Write data to "meters.xlsx" in current working directory
      - If verbose: print "downloaded {len} bytes from {url}" to stderr
      - Return nil on success

    - Use error type with exit code for clean error propagation. Options: custom struct implementing error, or sentinel errors with os.Exit in fetchCmd. Prefer a simple error wrapper struct that carries exit code.

    - Per D-16: error messages must be grep-able with prefixes FETCH_TIMEOUT, FETCH_DNS_FAILURE, FETCH_FAILED, FETCH_404, FETCH_5xx, FETCH_HTTP_*, FETCH_BAD_MAGIC, FETCH_TOO_SMALL

    - Per D-03: write to "meters.xlsx" always (no path flag)

    IMPORTANT: Do NOT use log.Fatal() -- per D-14 and RESEARCH.md anti-patterns, log.Fatal includes timestamps. Use fmt.Fprintf(os.Stderr, ...) and os.Exit(code).

    Per D-04: --verbose/-v flag controls stderr progress output. Silent by default.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI -> external HTTP server | Untrusted response crosses into the CLI when downloading xlsx from Google Sheets export URL or --url override |
| CLI -> local disk (meters.xlsx write) | The written file is consumed by the `build` subcommand in Plan 2 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | runFetch -> HTTP response | mitigate | Validate ZIP magic bytes (PK prefix) to distinguish actual xlsx from HTML login page or other non-xlsx payload |
| T-01-02 | Tampering | runFetch -> HTTP response | mitigate | Validate file size >= 10KB to detect truncated/corrupt downloads. TLS verifies HTTPS certificate chain via Go stdlib |
| T-01-03 | Information Disclosure | --url flag | accept | User controls the URL; flag is intentionally exposed for development/testing |
| T-01-04 | Denial of Service | runFetch -> disk write | accept | No upper bound on file size -- a malicious server could serve a large file. Mitigated by the fact that this is a developer CLI tool, not a service. Excelize handles large files gracefully for this use case. |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | No external Go dependency installs in this plan (excelize already in go.mod/go.sum). Package Legitimacy Audit in RESEARCH.md confirms all packages have [OK] disposition. |
</threat_model>
